# Архитектура проекта

app/
├── domain/           # Бизнес-логика (не зависит ни от чего)
│   ├── entities/     # Сущности (иногда еще называют model)
│   ├── usecases/     # бизнес-правила и операции (interactors)
│   └── repository/   # ИНТЕРФЕЙСЫ репозиториев
│
├── data/            # Реализации (зависит от domain)
│   ├── repository/  # РЕАЛИЗАЦИИ репозиториев
│   └── source/      # Источники данных (БД, API, файлы)
│
└── ui/    # UI (зависит от domain и data)

## Пометки себе

- Domain слой не зависит ни от чего - только чистый Kotlin 
- Нет Android/Desktop зависимостей в domain
- Use Case - это конкретное действие пользователя приложения. Каждый Use Case делает ОДНУ вещь.
